SMS Schedule Monitoring - Documentation

SMS Schedule Monitoring, a web application designed to provide real-time monitoring of the SMS processes status. Developed using React, Recharts, and FastAPI and accessible on http://10.145.8.23/.

FRONTEND

Technology Used -

  • React - React is a free and open-source front-end JavaScript library for building user interfaces based on components.
  • Recharts - Recharts is a chart library build with React and D3 that lets you deploy as declarative React components. The library provides native SVG support, lightweight dependency tree (D3 submodules) is highly customizable via component props.

Steps -

1. Install Node.js and npm -

  1. Download Node.js Installer:
    • Visit the official Node.js website at https://nodejs.org/.
    • Click on the "Downloads" button to access the download page.
    • Download the recommended LTS (Long-Term Support) version for Windows. This version includes both Node.js and npm.
  1. Run the Installer:
    • Once the installer is downloaded, double-click on the downloaded file to launch the installer.
    • Follow the prompts in the installer wizard. You can generally accept the default settings unless you have specific preferences.
    • Click "Next" to proceed through the installer.
    • Accept the license agreement and click "Next."
    • Choose the destination folder for installation or leave it as the default and click "Next."
    • Click "Install" to begin the installation process.
  1. Complete the Installation:
    • Wait for the installer to finish installing Node.js and npm on your system. This process may take a few moments.
    • Once the installation is complete, click "Finish" to exit the installer.
  1. Verify Installation:
    • To verify that Node.js and npm are installed correctly, open Command Prompt or PowerShell.
    • Type the following commands and press Enter after each one:
      node --version
npm --version

2. Run the existing React app -

  1. Navigate to the Project Directory:
    Open a terminal or command prompt and navigate to the directory where your React app is located. You can use the
    cd command to change directories. Example:
    cd path/to/your/react/app
  1. Install Dependencies :

    If the node_modules directory is not present in your project directory, you need to install dependencies. Run the following command:

    npm install

    This command will install all the dependencies listed in the package.json file.

  1. Start the Development Server:

    Once the dependencies are installed, you can start the development server by running:

    npm run dev

    This command will compile your React app and start a development server. You can visit http://localhost:5173/ or http://192.168.29.211:5173/ in your browser to view the app.

  1. View Your React App:
    Your React app should now be running locally. You can view and interact with it in your web browser. Any changes you make to the source code will automatically trigger a hot reload, meaning the browser will refresh to reflect your changes without needing to restart the server.
  1. Stop the Development Server:
    To stop the development server, you can press
    Ctrl + C in the terminal or command prompt where the server is running. This will shut down the server and stop the React app.

3. Deploying React App on an Nginx Server (RHEL) -

  1. Build Your React App:

    Before deploying, ensure your React app is built for production. Run the following command in your project directory:

    npm run build

    This command will create an optimized production build of your React app in the dist directory.

  1. Copy Build Files to Nginx HTML Directory:

    Copy the contents of the dist directory (created during the build process) to the Nginx HTML directory - /usr/share/nginx/html.

  1. Configure Nginx (if required):

    Navigate to the Nginx configuration directory:

    cd /etc/nginx/conf.d

    Open the configuration file - sms.conf :

    vim sms.conf

    Inside the configuration file, make the required changes.

    server {
    listen 80;
    server_name 10.145.8.23;

    location = /favicon.ico { access_log off; log_not_found off; }

    # Serve static files for FastAPI app
    location /static/ {
        root /home/sms/sms_backend;
    }

    # Proxy requests to FastAPI app
    location /api/schedule/ {
        proxy_pass http://0.0.0.0:8000/api/schedule/;
    }

    # Serve React app
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
  1. Test Nginx Configuration:

    Before restarting Nginx, it's a good idea to test the configuration for syntax errors:

    sudo nginx -t

    If there are no syntax errors, you'll see a message indicating that the configuration is okay.

  1. Restart Nginx:

    Once the configuration is validated, restart Nginx to apply the changes:

    sudo systemctl daemon-reload
sudo systemctl restart nginx
  1. Access the React app:

    You should now be able to access your React app in a web browser on http://10.145.8.23/.

BACKEND

Technology Used -

  • FastAPI - FastAPI is a web framework first released in 2018 for building RESTful APIs in Python.
  • Oracle Instant Client - Oracle Instant Client enables development and deployment of applications that connect to Oracle Database.

Steps -

1. Install Python and pip -

A. Install Python and pip on Windows -

  1. Download Python Installer:
    • Visit the official Python website at https://www.python.org/.
    • Click on the "Downloads" tab.
    • Choose the latest version of Python for Windows. Typically, it's prominently displayed on the page.
    • Click on the download link to get the installer.
  1. Run the Installer:
    • Once the installer is downloaded, double-click on the downloaded file to launch the installer.
    • Check the box that says "Add Python X.X to PATH" during the installation process. This option ensures that Python and pip are added to the system PATH, making them accessible from the command line.
    • Click "Install Now" to start the installation process
  1. Wait for Installation:
    • The installer will now install Python on your system. This process may take a few moments.
  1. Verify Python and pip Installation:
    • After the installation is complete, open Command Prompt or PowerShell.
    • Type the following command and press Enter:
      python --version
pip3 --version

B. Install Python and pip on Windows Subsystem for Linux (WSL) -

  1. Open WSL:
    Launch your preferred WSL distribution (e.g., Ubuntu) from the Start menu or by using the command line.
  1. Update Package Lists:

    Before installing new software, it's good practice to update the package lists to ensure you're getting the latest versions. Run the following command:

    sudo apt update
  1. Install Python:

    Python is usually pre-installed on most Linux distributions, but it's a good idea to ensure you have the latest version. Run the following command to install Python:

    sudo apt install python3

    This command will install Python 3, which is the latest version. You can verify the installation by checking the Python version:

    python3 --version
  1. Install pip:

    Pip is a package manager for Python that allows you to install and manage additional libraries and dependencies. To install pip, run the following command:

    sudo apt install python3-pip

    Once installed, you can verify the installation by checking the pip version:

    pip3 --version

    This should display the version of pip installed on your system.

2. Run the existing FastAPI app-

  1. Navigate to the Project Directory:
    Open a terminal or command prompt and navigate to the directory where your React app is located. You can use the
    cd command to change directories. Example:
    cd path/to/your/fastapi/app
  1. Create virtual environment:

    Open a terminal or command prompt and navigate to your project directory. Then, create a new virtual environment using the following command:

    python3 -m venv env
  1. Activate the Virtual Environment:

    Activate the virtual environment using the appropriate command for your operating system:

    • On Windows:
      env\Scripts\activate
    • On Linux:
      source env/bin/activate

    Activating the virtual environment ensures that any packages you install are isolated within the environment.

  1. Install Dependencies:

    If your FastAPI app has a requirements.txt file listing its dependencies, you can install them using pip. Run the following command:

    pip3 install -r requirements.txt

    This command installs all the dependencies listed in the requirements.txt file.

  1. Install Packages Using Wheels (if needed):

    If some dependencies have issues with installation via pip, you can try installing them using pre-compiled wheel files (.whl). First, ensure you have the wheel file for the package you want to install. Then, navigate to the directory containing the wheel file and run the following command:

    pip3 install --no-index wheels/*.whl
  1. Run the FastAPI App:

    Once all dependencies are installed, you can run your FastAPI app using the following command:

    uvicorn main:app --reload

    The --reload flag enables auto-reloading, allowing the server to restart automatically whenever changes are made to the code.

  1. Access Your FastAPI App:

    Open a web browser and navigate to the URL where your FastAPI app is running. By default, FastAPI apps are served at http://localhost:8000 .

3. Deploying FastAPI App on an Nginx Server (RHEL) -

  1. Copy FastAPI Application Files:

    Copy content of the FastAPI app files to the home directory - /home/sms/sms_backend except the env directory.

  1. Create virtual environment:

    Navigate to your project directory. Then, create a new virtual environment using the following command:

    python3 -m venv env
  1. Activate the Virtual Environment:

    Activate the virtual environment using following command:

    source env/bin/activate
  1. Install Dependencies using Wheels:

    First, ensure you have the wheel file for the package you want to install. Then run the following command:

    pip3 install --no-index wheels/*.whl
  1. Run the development server to test the setup:

    Run following command to test:

    gunicorn --workers 3 main:app --bind 10.145.8.23:8080 --worker-class uvicorn.workers.UvicornWorker --reload

    Open a web browser and navigate to the URL - http://10.145.8.23:8080/api/schedule

  1. Configure gunicorn (if required):

    Deactivate the virtual environment by executing the command below:

    deactivate

    Create a system socket file for gunicorn (if required) using following commands:

    cd /etc/systemd/system/
vim gunicorn.socket

    The file will contain the following instructions, make changes if necessary:

    [Unit]
Description=gunicorn socket

[Socket]
ListenStream=0.0.0.0:8000

[Install]
WantedBy=sockets.target

    Next, create a service file for gunicorn (if required) using following commands:

    vim gunicorn.service

    The file will contain the following instructions, make changes if necessary:

    [Unit]
Description=gunicorn daemon
Requires=gunicorn.socket
After=network.target

[Service]
User=sms
Group=root
WorkingDirectory=/home/sms/sms_backend
ExecStart=/home/sms/sms_backend/env/bin/gunicorn \
          --workers 3 \
          main:app \
          --bind 0.0.0.0:8000 \
          --worker-class uvicorn.workers.UvicornWorker

[Install]
WantedBy=multi-user.target

    After making necessary changes to the aforementioned files, run following commands:

    sudo systemctl daemon-reload
sudo systemctl restart gunicorn
  1. Configuring Nginx as a reverse proxy:

    Navigate to the Nginx configuration directory:

    cd /etc/nginx/conf.d

    Open the configuration file - sms.conf :

    vim sms.conf

    Inside the configuration file, make the required changes:

    server {
    listen 80;
    server_name 10.145.8.23;

    location = /favicon.ico { access_log off; log_not_found off; }

    # Serve static files for FastAPI app
    location /static/ {
        root /home/sms/sms_backend;
    }

    # Proxy requests to FastAPI app
    location /api/schedule/ {
        proxy_pass http://0.0.0.0:8000/api/schedule/;
    }

    # Serve React app
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
  1. Test Nginx Configuration:

    Before restarting Nginx, it's a good idea to test the configuration for syntax errors:

    sudo nginx -t

    If there are no syntax errors, you'll see a message indicating that the configuration is okay.

  1. Restart Nginx:

    Once the configuration is validated, restart Nginx to apply the changes:

    sudo systemctl daemon-reload
sudo systemctl restart nginx
  1. Access the FastAPI app:

    You should now be able to access your FastAPI app in a web browser on http://10.145.8.23/api/schedule.
